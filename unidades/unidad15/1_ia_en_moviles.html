25  Unidad 15: Inteligencia Artificial Móvil (Embedding)

Figura 1: Ecosistema TensorFlow Lite para inferencia on-device.

Este módulo es el puente hacia el desarrollo de aplicaciones de “Próxima Generación”. No vamos a usar solo APIs; vamos a aprender a enterrar la inteligencia dentro del dispositivo para que funcione sin internet, con baja latencia y máxima privacidad.

25.1 🎯 Objetivos de la Unidad

  • Dominar la arquitectura de IA “On-Device” frente a “Cloud-Based”.
  • Integrar el motor de inferencia de TensorFlow Lite en Flutter.
  • Gestionar modelos de visión y lenguaje localmente.
  • Optimizar el rendimiento mediante Cuantización y aceleración por Hardware.

25.2 📚 Lección 1: Arquitectura de IA en Móviles

La arquitectura on-device se diferencia radicalmente de las llamadas a API tradicionales. El modelo reside en la memoria del teléfono y utiliza el procesador local.

Diagrama 1: Pipeline de despliegue de modelos de IA en dispositivos móviles.

Cita Bibliográfica: > Abadi, M., et al. (2015). TensorFlow: Large-scale machine learning on heterogeneous systems. Software disponible en tensorflow.org. [1]

ImportantDesafío Anti-IA (Nivel 1)

Explica por qué un modelo de 70B de parámetros (como Llama 3) no puede correr en un iPhone 15 sin optimización extrema. ¿Es un problema de CPU, RAM o Almacenamiento?

25.3 📚 Lección 2: Integración de TensorFlow Lite

El estándar industrial para móviles. No vamos a usar un plugin mágico; vamos a entender el Interpreter.

Figura 2: Interacción entre el Intérprete y los Delegados de Hardware.

Cita Bibliográfica: > Howard, A. G., et al. (2017). MobileNets: Efficient Convolutional Neural Networks for Mobile Vision Applications. arXiv preprint arXiv:1704.04861. [2]

DART
// La clave está en el pre-procesamiento
// Los modelos no entienden "imágenes", entienden "matrices de floats"
final input = imagenNormalizada.reshape([1, 224, 224, 3]);
final output = List<double>.filled(1001, 0).reshape([1, 1001]);

// La ejecución es sincrónica, por eso usamos Isolates en producción
interpreter.run(input, output);
WarningDesafío Anti-IA (Nivel 2)

Implementa un Isolate en Dart para ejecutar la inferencia de TFLite. Si no lo haces, la UI de tu app se congelará durante el procesamiento. ¿Cómo pasarías los datos entre el hilo principal y el Isolate?

25.4 📚 Lección 3: Modelos de HuggingFace Localizados

HuggingFace es el “GitHub” de la IA. Aquí aprendemos a buscar modelos compatibles con TFLite o ONNX y cómo cargarlos localmente.

Importante: Si usas una API de inferencia remota, NO estás embebiendo el modelo. Aquí aprenderás a usar tflite_flutter para modelos locales.

TipDesafío Anti-IA (Nivel 3)

Encuentra un modelo de Análisis de Sentimiento en HuggingFace en formato .tflite y cárgalo como un Asset. Crea un mapper que convierta el índice numérico de salida (0, 1, 2) en un String legible (“Positivo”, “Neutral”, “Negativo”).

25.5 📚 Lección 4: Visión por Computadora (Object Detection)

Uso de la cámara en tiempo real. Aquí la optimización es crítica.

Conceptos clave: - FPS (Frames Per Second) vs Inferencia. - Cuantización (convertir pesos de 32 bits a 8 bits) para ganar velocidad.

25.6 📚 Lección 5: NLP y LLMs Ligeros

¿Podemos correr un chatbot local? Con modelos como Gemma-2b o Llama-Tiny, es posible. Aprenderás a configurar los límites de memoria para no crashear la app.

25.7 📚 Lección 6: Optimización de Modelos

Un modelo de 100MB es inaceptable para una app móvil. - Pruning: Quitar conexiones neuronales inútiles. - Quantization: Reducir la precisión sin perder (mucha) exactitud.

25.8 📚 Lección 7: IA en Clean Architecture

No ensucies tus Widgets con lógica de Tensores. - Data Source: El Motor de TFLite. - Repository: Maneja la conversión de Tipos Nativos a Tensores. - UseCase: La lógica de negocio (ej: “Si detectas fuego, lanza alerta”).

25.9 📚 Lección 8: Hibridación IA (Local + Cloud)

El mejor de los dos mundos. Procesamiento rápido local, análisis profundo en la nube si hay conexión.

25.10 🛠️ Herramientas Obligatorias

  • Netron: Para visualizar la arquitectura de tus modelos .tflite.
  • TensorFlow Hub: Catálogo oficial de modelos listos para usar.

25.11 🎓 Evaluación del Módulo

Para aprobar, el alumno debe entregar una app que: 1. Detecte al menos 5 tipos de objetos usando la cámara. 2. Realice la inferencia en menos de 500ms. 3. El código debe estar desacoplado de la UI mediante el patrón BLoC o Riverpod.

25.12 Final del Módulo

Recuerda: Cualquiera puede llamar a una API de OpenAI, pero solo un Ingeniero Senior sabe cómo meter un modelo dentro del bolsillo del usuario.

# Unidad 15: Inteligencia Artificial Móvil (Embedding)

![Logos de Tecnologías IA](../../images/tflite-logo.png){width=300}
*Figura 1: Ecosistema TensorFlow Lite para inferencia on-device.*

Este módulo es el puente hacia el desarrollo de aplicaciones de "Próxima Generación". No vamos a usar solo APIs; vamos a aprender a **enterrar** la inteligencia dentro del dispositivo para que funcione sin internet, con baja latencia y máxima privacidad.

## 🎯 Objetivos de la Unidad

- Dominar la arquitectura de IA "On-Device" frente a "Cloud-Based".
- Integrar el motor de inferencia de **TensorFlow Lite** en Flutter.
- Gestionar modelos de visión y lenguaje localmente.
- Optimizar el rendimiento mediante **Cuantización** y aceleración por Hardware.
